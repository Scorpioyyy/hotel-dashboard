# 002-RAG系统集成实施规划

## 项目概述

**目标**：将完整的 Python RAG 系统集成到 Next.js 项目中，替换当前简单的智能问答实现。

**关键约束**：
- ✅ **前端 UI 完全不变**（只修改后端逻辑）
- ✅ **禁用 HyDE 召回**（`enable_hyde=False`）
- ✅ **使用默认参数**（模型、权重、时效性）
- ✅ **Anaconda base 环境**（无需虚拟环境）

---

- **框架**：Next.js 15 + TypeScript

## 核心问题与解决方案

### Q1: Next.js 能否直接运行 Python 代码？
**答案**：❌ 不能

**解决方案**：FastAPI 后端服务
- Python RAG 系统封装为独立的 FastAPI 服务
- Next.js 通过 HTTP API 调用 Python 服务
- 部署：Next.js (Vercel) + Python (Railway)

### Q2: 是否需要安装 Python 环境？
**答案**：
- **开发环境**：使用 Anaconda base 环境
- **生产环境**：Railway 自动处理 Python 环境

### Q3: API Key 如何配置？
**答案**：环境变量配置

**开发环境** (`.env`)：
```bash
# Insforge
NEXT_PUBLIC_INSFORGE_BASE_URL=https://your-project.insforge.dev
NEXT_PUBLIC_INSFORGE_ANON_KEY=your_anon_key

# DashScope
DASHSCOPE_API_KEY=sk-xxxxx

# DashVector
DASHVECTOR_API_KEY=xxxxx
DASHVECTOR_HOTEL_ENDPOINT=xxxxx

# Python API (开发)
NEXT_PUBLIC_PYTHON_API_URL=http://localhost:8000
```

### Q4: 评论数据从哪里读取？
**答案**：✅ 从 Insforge 数据库读取

**原因**：
- 保持前后端数据一致性
- 支持数据动态更新
- 符合项目架构设计

### Q5: 倒排索引文件如何处理？
**答案**：存储在 Python 服务器文件系统

**原因**：
- `inverted_index.pkl` 是 Python pickle 文件
- JavaScript 无法读取
- 必须在 Python 环境中加载

### Q6: 是否需要拆分代码？
**答案**：✅ 必须拆分

**原因**：
- 当前 `lib.py` 有 1769 行代码
- 违反单一职责原则
- 难以维护和测试

### Phase 2: 数据源改造 🟠 P1

**目标**：评论数据从 CSV 改为 Insforge 数据库读取

**数据源映射**：
| 数据类型 | 当前来源 | 改造后 |
|---------|---------|--------|
| 评论数据 | CSV | Insforge 数据库 |
| 倒排索引 | PKL | 服务器文件系统 |
| 类别摘要 | ChromaDB | ChromaDB (不变) |
| 评论向量 | DashVector | DashVector (不变) |
| 反向Query | DashVector | DashVector (不变) |

**实现代码** (`utils/database.py`)：
```python
import os
import pandas as pd
import requests

def get_all_comments_from_insforge() -> pd.DataFrame:
    """从 Insforge 数据库获取所有评论"""
    base_url = os.getenv("NEXT_PUBLIC_INSFORGE_BASE_URL")
    anon_key = os.getenv("NEXT_PUBLIC_INSFORGE_ANON_KEY")

    headers = {
        "apikey": anon_key,
        "Authorization": f"Bearer {anon_key}",
        "Content-Type": "application/json"
    }

    all_data = []
    batch_size = 1000
    offset = 0

    while True:
        url = f"{base_url}/rest/v1/comments?select=*&offset={offset}&limit={batch_size}"
        response = requests.get(url, headers=headers)
        data = response.json()

        if not data:
            break

        all_data.extend(data)
        if len(data) < batch_size:
            break
        offset += batch_size

    df = pd.DataFrame(all_data).set_index("_id")
    return df
```

**当前接口** (不能改)：

**新实现**：
```typescript
const PYTHON_API_URL = process.env.NEXT_PUBLIC_PYTHON_API_URL || 'http://localhost:8000';

export async function getReferencesForQuestion(question: string): Promise<Comment[]> {
  const response = await fetch(`${PYTHON_API_URL}/api/v1/chat`, {
    method: 'POST',
    body: JSON.stringify({
      query: question,
      options: { enable_generation: false }  // 只检索
    })
  });

  const result = await response.json();
  return result.references.comments.map(convertToComment);
}

export async function* askQuestionStream(
  question: string,
  references: Comment[],
  signal?: AbortSignal
): AsyncGenerator<string, void, unknown> {
  const response = await fetch(`${PYTHON_API_URL}/api/v1/chat`, {
    method: 'POST',
    body: JSON.stringify({ query: question }),
    signal
  });

  const reader = response.body?.getReader();
  const decoder = new TextDecoder();

  while (true) {
    if (signal?.aborted) return;

    const { done, value } = await reader.read();
    if (done) break;

    const chunk = decoder.decode(value);
    const lines = chunk.split('\n\n');

    for (const line of lines) {
      if (line.startsWith('data: ')) {
        const event = JSON.parse(line.slice(6));
        if (event.type === 'chunk') {
          yield event.content;
        }
      }
    }
  }
}
```
